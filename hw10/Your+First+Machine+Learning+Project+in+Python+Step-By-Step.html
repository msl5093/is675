Data Collection and Preparation¶

The Iris data set contains 150 observations of four numeric features and one categorical factor, the species of the flower being observed. The relatively small size of the data set, and the easily interpreted scope of the features makes the Iris data set ideal for basic exploration of different machine learning models.

Python allows us to load the data directly from the UCI Machine Learning repository. We will also take this opporunity to import a number of the libraries that will be used during the assessment:

Model Fitting¶

Before fitting any algorithmic models, we will have to separate out a validation set for model fitting and improvement, and specify a sampling strategy for iterating models. We will follow a basic 80/20 split by using 80% of the data to train and fit models, leaving 20% for testing once a model has been selected. For resampling we will use a standard 10-fold cross-validation method. This will further split our train data set into 10 subsets, using nine for modeling fitting and one other for validation prior to turning the model loose on the unseen test data set.

In [38]:
# Split-out validation dataset
array = dataset.values
X = array[:,0:4]
Y = array[:,4]
validation_size = 0.20
seed = 7
X_train, X_validation, Y_train, Y_validation = cross_validation.train_test_split(X, Y, test_size=validation_size, random_state=seed)

# Test options and evaluation metric
num_folds = 10
num_instances = len(X_train)
seed = 7
scoring = 'accuracy'

In addition to subsetting our original data set and setting up our cross-validation parameters, we also set a variable (scoring) to a value of accuracy. This variable will be used to inform our models that accuracy should be used as the method for assessing model performance.

Accuracy simply refers to the number of correctly classified instances of the class variable that can be expressed as a raw percentage.

For modeling fitting we will be examining six different model types, logistic regression, linear discrimination analysis, k-nearest neighbors, classification and regression trees, Naive Bayes, and Support Vector Machines. A list of our model types is initialized as well as a list to hold results and the name of each model:

In [39]:
# Spot Check Algorithms
models = []
models.append(('LR', LogisticRegression()))
models.append(('LDA', LinearDiscriminantAnalysis()))
models.append(('KNN', KNeighborsClassifier()))
models.append(('CART', DecisionTreeClassifier()))
models.append(('NB', GaussianNB()))
models.append(('SVM', SVC()))
# evaluate each model in turn
results = []
names = []
for name, model in models:
	kfold = cross_validation.KFold(n=num_instances, n_folds=num_folds, random_state=seed)
	cv_results = cross_validation.cross_val_score(model, X_train, Y_train, cv=kfold, scoring=scoring)
	results.append(cv_results)
	names.append(name)
	msg = "%s: %f (%f)" % (name, cv_results.mean(), cv_results.std())
	print(msg)

A standard for loop allowed us to iterate over our list of models, fitting each one at a time. A seed value (which was set earlier, 7) ensures that each model is built using the exact same data splits to ensure that results of each can be directly comparable.

Our for loop fitted each model using the resampling method and subsets we specified earlier and printed the model name, accuracy, and the standard deviation for each model results.

It appears the k-Nearest Neighbors yielded the highest accuracy, although each model performed quite well during training and validation.
